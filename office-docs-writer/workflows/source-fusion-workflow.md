# 多源信息融合协议

## 适用场景

用户同时提供多种类型信息源时使用本协议进行统一处理。

## 信息源类型与处理方式

### 类型一：本地文件

用户提供文件路径（如 `[知识库根目录]/2024/xxx.md`）。

处理方式：
- 读取文件内容
- 提取与文档主题相关的事实、数据、表述
- 标注来源（"来自[文件名]"）

### 类型二：网页/在线文档

用户提供 URL 或复制的网页内容。

处理方式：
- 抓取或接收内容
- 识别权威表述（官方发布优先）
- 提取政策精神、数据、典型做法
- 标注来源（"来自[网站/文件名]"）

### 类型三：用户提供文本

用户直接在 prompt 中粘贴的原始材料。

处理方式：
- 视为一手信息，优先级高
- 提炼结构化内容
- 保留关键数据与原始表述

### 类型四：混合源

同时包含以上多种类型。

处理方式：
1. 分类整理各来源信息
2. 识别冲突或矛盾之处，向用户确认
3. 按文档逻辑结构分配各来源信息的覆盖段落
4. 起草时标注信息来源（可选，按用户要求）

## 信息缺口处理

当信息不足以支撑某个结构段落时：
- 在草稿中用 `【待补：[说明需要什么信息]】` 标注
- 不凭空补充数据或事实
- 告知用户需要哪些补充信息
