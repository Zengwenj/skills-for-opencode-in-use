# 专题会议记录信息源要求

## 必要信息（缺少则文档质量受损）

| 信息项 | 用途 | 获取方式 |
|-------|------|---------|
| 会议基本信息 | 会议头部 | 用户提供 |
| 上阶段工作部署文件 | 完成情况对应项 | 考核通知/工作计划 |
| 上阶段实际完成情况 | 第三部分主体 | 用户汇报或工作台账 |
| 本阶段工作安排 | 第六部分主体 | 考核通知/工作计划 |
| 学习传达材料 | 第五部分 | 上级文件/会议材料 |

## 辅助信息（有则质量更好）

| 信息项 | 用途 |
|-------|------|
| 问题台账/隐患记录 | 问题分析段落 |
| 上次会议记录 | 上阶段完成情况参考 |
| 领导发言提纲 | 会议要求段落 |
| 后续待办清单 | 第八部分 |

## 信息不足时的处理

- **上阶段完成情况不完整**：生成框架，用 `【待补：[具体工作事项完成情况]】` 标注缺失项
- **无学习传达材料**：第五部分仅生成结构，不填写传达内容
- **无后续待办清单**：第八部分标注 `【待补：请补充后续落实事项、责任人和时限】`
