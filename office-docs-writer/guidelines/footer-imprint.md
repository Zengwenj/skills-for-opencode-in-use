# 公文版记结构规范

> 本规范由 handoff-word格式调整记录.md (2026-04-30) 触发新增

## 适用范围

本规范仅覆盖中文公文末尾的三段式版记结构：

- **第 1 行**（表格）：联系人 + 联系电话
- **第 2 行**（表格）：印发单位 + 印发日期
- **第 3 行**（表格后独立段落）：（共印 X 份）

以上三段构成一个**特殊表格（2 行 × 1 列）+ 独立段落**的组合体，不是普通正文段落，也不是自由的 Markdown 表格。

## 不适用范围

本规范**不覆盖**以下内容：

- **抄送机关**、**密级**、**份号**、**紧急程度**等公文版头的其他排版形式
- 版记表格之外的**正文业务表格**（如统计表、明细表）
- 页眉、页脚、页码等页面级元素
- 发文机关标志、红色分隔线、发文字号等红头版头结构（参见 `formal-style.md`）

## 版记表格标准结构

### 参数表

| 参数 | 值 |
|------|-----|
| 表格行列 | 2 行 × 1 列 |
| 表格定位 | 以历史稿实测为准（存在 `w:tblpPr` 时复刻浮动定位，无 `tblpPr` 时保持内联流） |
| 竖线左右边框 | 无（`w:left`、`w:right` 均设为 `w:val="nil"`） |
| 横线上下边框 | 单实线（`w:top`、`w:bottom` 均设为 `w:val="single"`） |
| 表格外框 | 仅顶线（`w:tblBorders` 的 `w:top`） |
| 行距 | 固定值 28 磅（`w:lineRule="exact"`，`w:line="560"`） |
| 版记字体 | 以历史稿实测为准（无历史稿时经用户确认后可使用仿宋） |
| 字号 | 以历史稿实测为准（常见四号，对应 `w:sz w:val="32"`，仅供参考） |
| 共印份数位置 | 表格后的独立普通段落 |
| 段落间距 | 段前 0 行，段后 0 行 |

### 生成步骤概要

> 以下为生成步骤概要，非可执行代码。具体 OOXML 参数须按历史稿实测值填充。

1. 创建 2 行 × 1 列表格（`doc.add_table(rows=2, cols=1)`）
2. 设置 `w:tblPr` 表格属性：
   - `w:tblpPr`：浮动定位，设置 `w:leftFromText`、`w:rightFromText`、`w:vertAnchor`（`"text"`）、`w:horzAnchor`（`"page"`），以及 `w:tblpX`、`w:tblpY`（须按历史稿实测值填充）
   - `w:tblW`：表格宽度，`w:type="auto"`（或按历史稿类型）
   - `w:tblBorders`：仅顶线（`w:top` `w:val="single"`），其余边不设
3. 设置每个单元格的 `w:tcBorders`：
   - `w:top`、`w:bottom` → `w:val="single"`（`w:sz`、`w:space`、`w:color` 按历史稿）
   - `w:left`、`w:right` → `w:val="nil"`
4. 第 1 行写入联系人信息：
   - 设置 `w:rFonts` 的 `w:eastAsia` 为版记字体（以历史稿实测为准）
   - 设置 `w:sz` 字号（以历史稿实测为准）
   - 设置 `w:ind` 首行缩进（以历史稿实测为准）
5. 第 2 行写入印发单位/印发日期：
   - 同第 1 行设置字体、字号、缩进
6. 表格后追加独立段落写入共印份数：
   - 字体、字号、缩进同第 1 行

> **以上步骤概要为参考结构，执行代理须根据历史稿参数化以下变量**：
> `w:tblpX`、`w:tblpY`、`w:gridCol` 列宽、`w:tcW` 单元格宽度、`w:line` 行距值、
> `w:firstLineChars` 缩进值、`w:rFonts` 字体名称。不可将参考值固化为默认值。

### 关键说明

**表格结构**：
- ❌ 错误：将联系人、印发单位写入普通正文段落
- ❌ 错误：版记表格设置 `w:left`、`w:right` 为非 `nil` 值（出现可见竖线）
- ❌ 错误：共印份数放入表格单元格内
- ✅ 正确：联系人/印发单位位于 2 行 × 1 列表格中，表格仅横线无竖线，共印份数在表格之后作为独立段落

**字体独立**：
- ❌ 错误：版记表格直接继承正文段落样式，字体与正文完全相同
- ✅ 正确：版记字体通过 `w:eastAsia` 独立设置，以历史稿实测为准

**生成顺序**：
- ✅ 正确：正文末段 → 版记表格 → 共印份数段落 → sectPr
- ❌ 错误：共印份数出现在表格之前，或表格出现在正文中间

## OOXML XML 参考片段

以下 XML 片段均提取自历史发文稿，标注为参考结构，**执行代理须根据历史稿参数化后使用**。

### 表格浮动定位（w:tblpPr）

```xml
<!-- leftFromText、rightFromText 为示例值，须按历史稿替换 -->
<!-- vertAnchor、horzAnchor 为示例值，须按历史稿替换 -->
<!-- tblpX、tblpY 为示例值，须按历史稿替换 -->
<w:tblpPr w:leftFromText="180" w:rightFromText="180"
          w:vertAnchor="text" w:horzAnchor="page"
          w:tblpX="1689" w:tblpY="546"/>
<w:tblOverlap w:val="never"/>
```

### 表格整体边框（w:tblBorders）

```xml
<!-- 仅顶线，无底线、左线、右线；w:sz、w:space、w:color 为示例值，须按历史稿替换 -->
<w:tblBorders>
  <w:top w:val="single" w:sz="12" w:space="0" w:color="auto"/>
</w:tblBorders>
```

### 单元格边框（w:tcBorders）

```xml
<!-- 左右两侧设为 nil，确保无可见竖线 -->
<w:tcBorders>
  <w:top w:val="single" w:sz="12" w:space="0" w:color="auto"/>
  <w:left w:val="nil"/>
  <w:bottom w:val="single" w:sz="12" w:space="0" w:color="auto"/>
  <w:right w:val="nil"/>
</w:tcBorders>
```

### 单元格宽度与网格

```xml
<!-- 列宽为示例值，须按历史稿替换 -->
<w:tblGrid>
  <w:gridCol w:w="9208"/>
</w:tblGrid>
<w:tcW w:w="9208" w:type="dxa"/>
```

### 版记字体与字号

```xml
<!-- "仿宋" 字体名称为示例值，w:sz w:val="32" 为示例值，均须按历史稿实测替换 -->
<w:rFonts w:ascii="仿宋" w:eastAsia="仿宋" w:hAnsi="仿宋" w:cs="仿宋"/>
<w:sz w:val="32"/>
```

### 单元格段落格式

```xml
<!-- w:line 和 w:firstLineChars 为示例值，须按历史稿替换 -->
<w:spacing w:line="560" w:lineRule="exact"/>
<w:ind w:rightChars="-106" w:right="-223" w:firstLineChars="50" w:firstLine="160"/>
```

## 字体差异说明

版记字体与正文主字体是两套独立设置，二者允许差异：

| 区域 | 字体 | 依据 |
|------|------|------|
| 正文 | 方正仿宋_GBK | `guidelines/formal-style.md` 字体规范 |
| 版记表格 | 仿宋 | 历史稿实测（非方正仿宋_GBK） |

**处置原则**：

1. **有历史稿时**：版记字体以历史稿 OOXML 中 `w:eastAsia` 的实测值为准，不对历史稿字体做二次修正。
2. **无历史稿时**：版记默认使用**仿宋**（非方正仿宋_GBK），不得机械套用正文样式中的正文字体。
3. **版本注意**：即使正文整体使用方正仿宋_GBK，版记表格仍需独立设置字体，不得继承或依赖全局样式。

## 生成链能力前置检查

在生成包含版记表格的 DOCX 之前，必须确认生成链支持以下能力：

| 检查项 | 说明 |
|--------|------|
| 浮动表格定位（`w:tblpPr`） | 确认生成链可将表格定位到页面底部（`w:vertAnchor="text"`、`w:horzAnchor="page"`）；若生成链不支持浮动定位，须显式说明风险，不得静默回退到普通表格 |
| 单元格边框独立控制（`w:tcBorders`） | 确认生成链可逐个设置 top/left/bottom/right 边框，且 `w:val="nil"` 能被正确写入 XML |
| 中文字体 `w:eastAsia` | 确认生成链可设置区别于正文的独立中文字体 |
| 表格行列顺序 | 确认版记表格位于正文末段之后、共印份数段落之前 |
| 表格后独立段落 | 确认生成链可在表格之后追加普通段落（不嵌套在表格内） |

若任一检查项不满足，须在输出中显式标注风险，不得仅以视觉截图作为验证依据。

## 失败信号清单

以下任一信号出现，即表示版记生成失败：

1. **联系人出现在普通正文段落中**：联系人信息未进入表格，而是以 `w:p` 形式出现在正文区域。
2. **版记区域无表格结构**：文档末尾无 `w:tbl` 元素，仅以段落模拟版记外观。
3. **版记表格有可见竖线**：`w:tcBorders` 中 `w:left` 或 `w:right` 为非 `nil` 值，Word 渲染出左右竖线。
4. **版记字体与正文完全相同**：版记表格的 `w:eastAsia` 值与正文段落完全相同，未做独立设置。
5. **共印份数位于表格单元格内**：`（共印 X 份）` 被写入表格的第 3 行或与印发单位合并在同一单元格。
6. **未检查历史稿就写入版记**：在存在历史 DOCX 参考的情况下，直接按默认参数生成版记，未读取历史稿的 OOXML 结构。
7. **浮动定位静默回退**：生成链不支持 `w:tblpPr` 浮动定位，但未报告风险，直接生成普通表格靠左对齐。

## Markdown 表格分类处理表

公文生成脚本中遇到 Markdown 表格时，须按内容类型分类处理：

| 表格内容 | 建议处理 |
|----------|----------|
| 联系人 / 联系电话 / 印发单位 | 转为版记表格（浮动定位，仅横线，独立字体） |
| 正文业务表格（统计数据、明细、对比表等） | 转为 Word 普通表格（保留原表格行列和边框） |
| 空表格或仅作分隔线使用 | 跳过，不生成任何表格元素 |
| 未识别内容的表格 | 保守保留为 Word 表格，并提示人工检查是否需要转入版记 |
