# 上下文变量命名规范

## 用途

本文件定义所有文档模板中使用的占位符变量，确保命名统一。
生成文档时，这些变量从用户提供的上下文中获取，不硬编码。

## 变量清单

### 组织类

| 变量名 | 说明 | 示例值 |
|-------|------|--------|
| `[机构名称]` | 当前机构全称 | XX公司、XX支行 |
| `[上级机构]` | 上级组织名称 | 总公司、分行 |
| `[上级党委]` | 上级党委全称 | 党委 |
| `[部门名称]` | 当前部门或支部名称 | 党支部、纪检部 |
| `[单位简称]` | 机构缩写，用于标题 | 根据机构名称自然缩写 |

### 时间类

| 变量名 | 说明 | 示例值 |
|-------|------|--------|
| `[年份]` | 四位数年份 | 2024、2025 |
| `[季度]` | 季度表述 | 一季度、上半年 |
| `[考核周期]` | 工作考核周期描述 | 1月至3月、第一季度 |
| `[会议日期]` | 具体会议日期 | 2025年3月15日 |

### 政策类

| 变量名 | 说明 |
|-------|------|
| `[当前重要会议精神]` | 当前有效的重要会议和政策精神（不固定到特定届次） |
| `[上级重要部署]` | 上级党委的近期重要工作部署 |
| `[考核办法名称]` | 具体考核办法或细则的名称 |

### 路径类

| 变量名 | 说明 |
|-------|------|
| `[知识库根目录]` | 本地知识库根目录路径 |
| `[材料存放路径]` | 具体材料文件的存放路径 |

## 使用规则

1. 模板文件中一律使用 `[变量名]` 格式
2. 生成文档前，agent 应主动确认用户已提供的关键变量
3. 用户未提供时，在输出文档中保留 `[变量名]` 格式，不推测填写
4. 禁止在模板文件中出现具体的机构名称、年份、路径

## 新增变量

如业务需要新增变量，应：
1. 先在本文件中定义
2. 再在具体模板中使用
3. 记录到 maintenance/change-log.md
