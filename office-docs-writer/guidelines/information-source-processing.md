# 信息源处理规范

## 概述

本文件规定从不同类型信息源提取内容时的处理方法。
所有路径、机构名称、年份均使用上下文变量，不硬编码。

## 信息源类型

### 一、本地知识库文件

**来源**：[知识库根目录] 下的本地文档（Word、Markdown、PDF 等）

**处理步骤**：
1. 根据用户提供的路径读取文件
2. 识别文件类型：考核通知 / 工作材料 / 会议纪要 / 上级文件
3. 提取与目标文档相关的：
   - 关键数据与指标
   - 工作部署与要求
   - 进展与完成情况
   - 经验做法与典型案例
4. 保留原文关键表述，不随意改写政治性内容

**注意**：
- 读取失败时，告知用户并跳过，不编造文件内容
- 如路径包含敏感信息，输出中省略具体路径

### 二、网页与在线文档

**来源**：用户提供的 URL，或公开政府/监管网站

**处理步骤**：
1. 抓取页面内容
2. 识别权威级别：党政机关官网 > 行业协会 > 新闻媒体
3. 提取政策要求、数据、典型表述
4. 标注内容来源（网站名称 + 发布时间）

**注意**：
- 网页内容可能过期，注意核对发布日期
- 不使用来源不明或未经核实的信息

### 三、用户直接提供的文本

**来源**：用户在 prompt 中粘贴的原始材料

**处理步骤**：
1. 视为一手信息，优先级最高
2. 识别内容类型：会议记录 / 工作汇报 / 考核通知 / 学习材料
3. 结构化提炼后用于起草
4. 对原文中的数字、人名、事件保持准确

### 四、混合信息源

**处理步骤**：
1. 分类整理（按上述三类）
2. 识别信息冲突：同一事项多源描述不一致时，告知用户确认
3. 按文档逻辑结构分配各来源的使用段落
4. 起草时内部标注来源（不在最终文档中出现，除非用户要求）

## 信息缺口规则

当信息不足以支撑某段落时：
- 使用 `【待补：[说明】` 标注
- 不凭空补充未经提供的数据或事实
- 在交付时告知用户需要补充的内容

## 路径与隐私

- 所有文件路径在输出中仅保留文件名，不暴露完整路径
- 人员姓名、账号、联系方式等个人信息，按用户要求处理（保留 / 脱敏 / 省略）
